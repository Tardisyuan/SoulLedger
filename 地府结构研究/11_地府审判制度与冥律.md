# 地府审判制度与冥律详解

地府的审判制度是一套完整的司法体系，依据《冥律》（又称《阴律》《玉历宝钞》）对亡魂进行审理、判决和刑罚执行。

---

## 一、冥律概述

### 1.1 冥律的定位

| 项目 | 内容 |
|------|------|
| 别称 | 阴律、玉历宝钞、冥律 |
| 性质 | 地府的法律法规体系 |
| 核心原则 | 赏善罚恶，一秉至正至公 |
| 上位法 | 天律（天道的法律） |
| 适用范围 | 所有亡魂及阴间神祇 |

### 1.2 冥律的来源

冥律据传由天庭颁布，玉帝审核后下达酆都大帝执行。主要经典依据：
- **《玉历宝钞》** — 地府审判的核心依据
- **《地藏十王经》** — 十殿阎王职责的依据
- **《太上老君律》** — 早期道教冥律
- **《太乙救苦天尊经》** — 度化亡魂的依据

### 1.3 冥律的基本原则

1. **因果报应**：善有善报，恶有恶报
2. **功过相抵**：善行可减轻恶行带来的刑罚
3. **寿数既定**：寿命由生死簿注定，不得擅自增减
4. **审判公正**：有起诉、辩护、判决、上诉的完整程序
5. **刑罚适当**：刑罚与罪行相匹配

```
         ┌─────────────────┐
         │    天庭（玉帝）  │  ← 上位法来源
         └────────┬────────┘
                  │ 颁布冥律
                  ▼
         ┌─────────────────┐
         │    酆都大帝      │  ← 阴间最高行政+司法长官
         └────────┬────────┘
                  │
                  ▼
         ┌─────────────────┐
         │    十殿阎王      │  ← 各殿审判机构
         │  （实质审判权）   │
         └────────┬────────┘
                  │
                  ▼
         ┌─────────────────┐
         │    四大判官      │  ← 协助审理
         │    六案功曹      │  ← 文书记录
         └────────┬────────┘
                  │
                  ▼
         ┌─────────────────┐
         │   地方城隍       │  ← 初审/地方受理
         └─────────────────┘
```

## 三、审判流程

### 3.1 完整审判流程

```
亡魂进入酆都城
        │
        ▼
【第一步：立案登记】
城隍提交案件 → 六案功曹录入 → 创建案件档案
        │
        ▼
【第二步：初步审查】
崔珏核查生死簿 → 核实生平记录 → 确认罪行清单
        │
        ▼
【第三步：起诉】
由六曹之一（通常为鬼曹）提出起诉 → 列明罪状
        │
        ▼
【第四步：分配审判殿】
根据罪行类型分配至对应殿位（初判殿或复审殿）
        │
        ▼
【第五步：开庭审判】
阎王审阅案卷 → 亡魂陈述 → 证人作证 → 证据质证
        │
        ▼
【第六步：一审判决】
判决结果：guilty（定罪）/ innocent（无罪）/ partial（部分成立）
        │
        ├─── 无罪 → 直接投胎
        │
        └─── 有罪 → 量刑
        │
        ▼
【第七步：量刑】
根据冥律对应条款 → 决定刑罚类型 → 分配地狱层数
        │
        ▼
【第八步：上诉（可选）】
不服判决 → 向上一级阎王申诉 → 魏征核查 → 重新审理
        │
        ▼
【第九步：刑罚执行】
分配至对应地狱 → 执行刑罚 → 记录服刑进度
        │
        ▼
【第十步：刑满释放】
刑罚执行完毕 → 释放至轮回殿
        │
        ▼
【第十一步：最终裁定】
转轮王核定 → 决定投胎道别/身份
        │
        ▼
【第十二步：孟婆汤 + 投胎】
喝孟婆汤 → 发放转世批文 → 进入六道轮回
```

### 3.2 上诉制度

| 上诉条件 | 说明 |
|---------|------|
| 理由 | 冤屈、误判、证据不足 |
| 上诉次数 | 通常最多3次 |
| 上诉路径 | 本殿阎王 → 上一殿阎王 → 酆都大帝 |
| 审核机构 | 魏征（察查司）负责稽查冤假错案 |
| 改判条件 | 证据确证原判有误 |

### 3.3 特殊情况审理

**直接投胎（不经各殿）**：
- 大善人（善行积分≥500）
- 无罪之人（生平无大恶）
- 冤死但寿数已尽

**直接入阿鼻地狱（不经各殿）**：
- 犯上品五逆十恶（杀父、杀母、杀罗汉、出佛身血、破和合僧）
- 诽谤正法且不知悔改

## 四、冥律罚则详解

### 4.1 十恶罪行与对应刑罚

| 十恶 | 具体行为 | 刑罚地狱 | 备注 |
|------|---------|---------|------|
| 杀生 | 故意杀害人/动物 | 刀山地狱 | 杀生越多刑罚越重 |
| 偷盗 | 偷窃抢劫 | 牛坑地狱 | 返还赃物无效 |
| 邪淫 | 卖淫嫖娼 | 油锅地狱 | 婚内正常性行为不计 |
| 妄语 | 撒谎骗人 | 拔舌地狱 | 两舌（挑拨离间）加重 |
| 饮酒 | 醉酒乱性 | 烊铜地狱 | 道教有禁酒令 |
| 邪见 | 诽谤正法 | 阿鼻地狱 | 最重罪行之一 |

### 4.2 十善功德与奖励

| 十善 | 具体行为 | 功德积分 |
|------|---------|---------|
| 孝养父母 | 赡养双亲 | +100 |
| 戒杀放生 | 不杀生且放生 | +50 |
| 布施行善 | 施舍济困 | +50 |
| 诚信不欺 | 不说谎守信 | +30 |
| 救死扶伤 | 帮助危难之人 | +40 |
| 敬奉神明 | 正信神明 | +10 |
| 持守五戒 | 不杀不盗不淫不妄不酒 | +80 |

### 4.3 功过相抵计算

冥律中功过相抵的计算规则：

| 条件 | 结果 |
|------|------|
| 功德 > 罪过 | 可减轻刑罚或免于入狱 |
| 功德 = 罪过 | 入狱但可减刑 |
| 功德 < 罪过 | 按差额执行相应刑罚 |
| 功德 >> 罪过 | 可直接投胎善道 |

## 五、证据制度

### 5.1 证据类型

| 证据类型 | 来源 | 可信度 |
|---------|------|--------|
| 生死簿记录 | 崔珏/功曹实时记录 | 最高 |
| 证人证词 | 城隍/土地公/日游神/夜游神 | 高 |
| 现场勘查 | 黑白无常/牛头马面 | 中高 |
| 悔过书 | 亡魂自述 | 参考 |
| 三生石显现 | 前世记忆 | 辅助参考 |

### 5.2 证据采集

- **生前行为记录**：由日游神/夜游神实时记录
- **临终遗言**：由土地公转交
- **遗物佐证**：烧纸/祭祀记录（可证明孝心）
- **目击证人**：相关亡魂/神祇

## 六、冥律与天律的关系

| 对比项 | 冥律 | 天律 |
|--------|------|------|
| 适用范围 | 亡魂及阴间 | 神仙及天道 |
| 惩罚形式 | 地狱刑罚 | 天劫/雷劈/削去神籍 |
| 执行者 | 阎王/判官 | 天庭执行机构 |
| 上诉途径 | 本系统内申诉 | 向玉帝申诉 |
| 时间跨度 | 跨越多世 | 即时报应 |

## 七、审判制度管理系统建模

### 7.1 核心数据表

#### 审判案件表（详细版）(trial_case)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| case_number | VARCHAR(50) | 案号（格式：殿ID-年份-序号） |
| soul_id | BIGINT FK | 亡魂ID |
| plaintiff_type | ENUM | 起诉方（鬼曹/城隍/功曹） |
| charges | JSON | 罪状列表 |
| evidence | JSON | 证据列表 |
| defense_statement | TEXT | 亡魂陈述 |
| witness_records | JSON | 证人证词 |
| verdict | ENUM | guilty/innocent/partial |
| verdict_details | JSON | 判决详情 |
| sentence | JSON | 判决（地狱/释放/投胎） |
| appeal_count | INT | 上诉次数 |
| appeal_records | JSON | 上诉记录 |
| case_status | ENUM | filed/investigating/trial/judged/appealed/closed |
| judge_id | BIGINT FK | 审判阎王ID |
| recorder_id | BIGINT FK | 记录功曹ID |

#### 冥律条款表 (hell_law)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| article_number | VARCHAR(20) | 条款编号 |
| sin_category | VARCHAR(50) | 罪责类别 |
| description | TEXT | 条款描述 |
| applicable_hell | INT | 适用地狱层数 |
| min_punishment_years | INT | 最短刑期（年） |
| max_punishment_years | INT | 最长刑期（年） |
| mitigating_factors | JSON | 减轻情节 |
| aggravating_factors | JSON | 加重情节 |

### 7.2 API接口补充

```markdown
### 审判管理
- POST /api/v1/cases              # 创建案件（立案）
- GET  /api/v1/cases/{id}         # 获取案件详情
- PUT  /api/v1/cases/{id}/verdict # 判决
- POST /api/v1/cases/{id}/appeal  # 上诉
- GET  /api/v1/cases/{id}/appeals # 查看上诉记录

### 冥律管理
- GET  /api/v1/laws               # 获取冥律条款
- GET  /api/v1/laws/{article}      # 获取特定条款
- POST /api/v1/laws/calculate     # 计算量刑（输入罪行，返回地狱/刑期）

### 证据管理
- POST /api/v1/cases/{id}/evidence # 添加证据
- GET  /api/v1/cases/{id}/evidence # 查看证据清单
```
