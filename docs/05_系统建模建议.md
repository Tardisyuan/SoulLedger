### 1.2 核心数据库表结构

#### 1.2.1 神祇表 (deity)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| name | VARCHAR(50) | 神祇名称 |
| title | VARCHAR(100) | 职位称号 |
| rank | TINYINT | 级别(1-5) |
| deity_type | ENUM | '最高神祇','鬼帝','阎王','判官','功曹','鬼差','地方神' |
| parent_id | BIGINT FK | 上级神祇ID |
| department_id | BIGINT FK | 所属部门ID |
| authority | JSON | 职权范围 |
| location | VARCHAR(100) | 所在位置 |
| description | TEXT | 描述 |
| status | TINYINT | 状态(1在职/0离任) |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 1.2.2 亡魂表 (soul)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| name | VARCHAR(50) | 亡魂姓名 |
| gender | TINYINT | 性别(1男2女) |
| birth_date | DATE | 出生日期 |
| death_date | DATETIME | 死亡时间 |
| death_cause | VARCHAR(100) | 死因 |
| birth_place | VARCHAR(200) | 出生地 |
| death_place | VARCHAR(200) | 死亡地点 |
| current_status | ENUM | 亡魂状态 |
| current_location | VARCHAR(100) | 当前位置(十三站之一) |
| sin_score | INT | 罪恶值(负数为善，正数为恶) |
| good_deeds | JSON | 善行记录 |
| bad_deeds | JSON | 恶行记录 |
| registered_at | DATETIME | 入籍时间 |
| district_id | BIGINT FK | 所属城隍ID |

**亡魂状态枚举:**
- `PENDING_REGISTRATION` 待登记
- `ON_HUANGQUAN_ROAD` 黄泉路中
- `AT_WANGXIANG_TAI` 望乡台
- `PASSING_DANGER_ZONES` 过恶狗岭/金鸡山
- `AT_MIHUN_DIAN` 迷魂殿
- `IN_FENGDU` 酆都城
- `UNDER_TRIAL` 审判中
- `SERVING_PUNISHMENT` 服刑中
- `AT_GHOST_VILLAGE` 野鬼村
- `AT_LIANHUA_TAI` 莲花台
- `AT_HUANHUN_CLIFF` 还魂崖
- `REINCARNATED` 已投胎

#### 1.2.3 生死簿表 (life_death_record)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| soul_id | BIGINT FK UNIQUE | 亡魂ID(一人一条记录) |
| original_lifespan | INT | 原本寿命(天) |
| remaining_days | INT | 剩余寿命 |
| good_score | INT | 善行总分 |
| bad_score | INT | 恶行总分 |
| record_book_code | VARCHAR(50) | 簿册编号 |
| page_number | INT | 页码 |
| entry_number | INT | 条目号 |
| recorded_by | BIGINT FK | 记录官(崔珏或功曹) |
| last_modified | DATETIME | 最后修改时间 |
| status | ENUM | 'active','closed','sealed' |

#### 1.2.4 审判案件表 (trial_case)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| case_number | VARCHAR(50) UNIQUE | 案号(格式:殿ID-年-序号) |
| soul_id | BIGINT FK | 亡魂ID |
| court_id | BIGINT FK | 受理殿ID |
| current_judge_id | BIGINT FK | 当前审判官 |
| charges | JSON | 罪状列表 |
| evidence | JSON | 证据列表 |
| witness_records | JSON | 证人证词 |
| verdict | ENUM | 'guilty','innocent','partial' |
| sentence | JSON | 判决(刑罚/释放/投胎) |
| appeal_count | INT | 上诉次数 |
| appeal_result | JSON | 上诉结果 |
| case_status | ENUM | 'filed','investigating','trialing','judged','appealed','closed' |
| filed_at | DATETIME | 立案时间 |
| judged_at | DATETIME | 判决时间 |
| closed_at | DATETIME | 结案时间 |

#### 1.2.5 刑罚执行表 (punishment_record)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| case_id | BIGINT FK | 关联案件ID |

| actual_end_date | DATETIME | 实际结束时间 |

| execution_status | ENUM | 'pending','executing','paused','completed' |

#### 1.2.6 投胎记录表 (reincarnation_record)

| 字段 | 类型 | 说明 |

## 二、业务流程建模

### 2.1 亡魂处理完整流程

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                           亡魂生命周期管理                                    │
└──────────────────────────────────────────────────────────────────────────────┘

[死亡]
    │
    ▼
[1. 土地庙登记] ──→ 创建soul记录 ──→ 城隍初审
    │
    ▼
[2. 黄泉路押送] ──→ 黑白无常/牛头马面押送
    │
    ▼
[3. 望乡台眺望] ──→ 亡魂最后告别 ──→ 记录心愿
    │
    ▼
[4. 恶狗岭/金鸡山] ──→ 考验通过 ──→ 记录善恶
    │
    ▼
[5. 迷魂殿] ──→ 服用迷魂水 ──→ 清醒头脑
    │
    ▼
[6. 酆都城立案] ──→ 创建trial_case ──→ 分配至各殿
    │
    ├──→ 第一殿(秦广王)初判
    │       │
    │       ├──→ 善人 ──→ 直接批准投胎
    │       │
    │       └──→ 恶人 ──→ 移交下一殿
    │
    ├──→ 第二殿(楚江王)
    ├──→ 第三殿(宋帝王)
    ├──→ 第四殿(五官王)
    ├──→ 第五殿(阎罗王) ──→ 复审
    ├──→ 第六殿(卞城王)
    ├──→ 第七殿(泰山王)
    ├──→ 第八殿(都市王)
    ├──→ 第九殿(平等王) ──→ 阿鼻地狱
    │
    ▼
[7. 第十殿(转轮王)] ──→ 最终裁定 ──→ 决定投胎方向
    │
    ▼
[8. 供养阁] ──→ 处理纸钱祭祀
    │
    ▼
[9. 莲花台] ──→ 地藏王度化
    │
    ▼
[10. 还魂崖] ──→ 孟婆汤 ──→ 记忆消除
    │
    ▼
[11. 投胎转世] ──→ 创建reincarnation_record ──→ 新生命诞生
```

### 2.2 勾魂业务流程

```
城隍/阎王下令勾魂
        │
        ▼
黑白无常接单 ──→ 前往阳间
        │
        ▼
找到目标 ──→ 出示拘魂令
        │
        ▼
亡魂脱离肉体 ──→ 佩戴枷锁
        │
        ▼
押送至土地庙 ──→ 登记
        │
        ▼
送往黄泉路 ──→ 任务完成
```

## 四、API接口设计(建议)

### 4.1 亡魂管理
- `POST /api/v1/souls` - 登记新亡魂
- `GET /api/v1/souls/{id}` - 获取亡魂信息
- `PUT /api/v1/souls/{id}/location` - 更新亡魂位置
- `GET /api/v1/souls?status=xxx&location=xxx` - 条件查询

### 4.2 审判管理
- `POST /api/v1/cases` - 创建案件
- `GET /api/v1/cases/{id}` - 获取案件详情
- `PUT /api/v1/cases/{id}/verdict` - 判决
- `POST /api/v1/cases/{id}/appeal` - 上诉

### 4.3 刑罚执行
- `GET /api/v1/punishments/hell/{layer}` - 查看某层地狱
- `PUT /api/v1/punishments/{id}/progress` - 更新执行进度

### 4.4 投胎管理
- `POST /api/v1/reincarnations` - 创建投胎记录
- `GET /api/v1/reincarnations/types` - 获取六道类型
